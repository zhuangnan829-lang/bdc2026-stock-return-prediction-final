# 一页实验结论摘要

## 课题目标

面向沪深300成分股历史行情数据，构建“数据预处理 - 特征工程 - 收益预测 - 选股排序 - 组合回测”的完整闭环，输出符合比赛要求的 `result.csv`，并重点优化交易成本后的组合收益表现。

## 当前已经完成的内容

- 完成原始数据整理、字段统一、时间切分和 `data_manifest`
- 完成标签构造，严格对应 `T+1` 开盘买入、`T+5` 开盘卖出
- 完成特征工程主线版本
- 完成 `LightGBM` 排序基线训练
- 完成 `walk-forward` 验证
- 完成风险过滤、风险调整排序、权重分配和低换手执行
- 完成本地回测、系统化消融、结果校验和提交前检查
- 完成 Docker 离线复现实验

## 关键实验结论

### 1. 特征结论

- `base_technical_risk` 是当前最优特征集
- 后续新增“相对强弱 + 连续涨跌 + 波动状态切换”特征后，未能超过当前默认方案，因此未采纳

### 2. 排序策略结论

- `risk_adjusted` 优于 `pure_prediction`

### 3. 权重策略结论

- 早期独立权重消融里，`risk_adjusted` 更优
- 在固定最新候选池和风险阈值后，联合搜索显示 `pred` 更优，因此当前默认权重已切换到 `pred`

### 4. 换手控制结论

- `max_turnover` 从 `0.5` 到 `0.75` 都做过验证
- 以“收益率 + 稳定性平衡”为目标，当前默认选择 `max_turnover=0.70`

## 当前最终默认方案

当前冻结默认提交方案为：

`base_technical_risk + 风险调整排序 + 预测值权重 + 平衡版低换手执行（candidate_size=55, risk_penalty=0.20, max_turnover=0.70）`

## 当前阶段结论

当前程序已经从“能跑通 baseline”推进到“可训练、可回测、可消融、可 Docker 离线复现、可提交校验”的稳定阶段。  
程序主线已经基本完成，后续重点应放在最终汇报材料整理、结果快照固化以及答辩展示载体建设上。
