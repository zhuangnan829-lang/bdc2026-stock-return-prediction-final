## 七、如果目标是“提高程序收益率”，下一步应该怎么做

这一部分不再从“是否完成模块”出发，而是专门从“收益率还能怎么抬”出发给建议。

结论先说：

> 你当前程序要继续提高收益率，最有希望的方向不是大改主线，而是围绕“标签更贴近目标、验证更稳、推理更精细、换手更可控、候选池更干净”五个方向做增量优化。

### 1. 优先方向一：进一步优化排序目标，而不是只改模型

当前状态：

- 你现在已经用 `cross_section_rank` 训练目标，方向是对的

下一步可尝试：

- 增加按日期横截面 z-score 的目标对照
- 对未来收益做截尾 / winsorize，再做排序目标
- 尝试把训练目标做成更强调 Top-K 的版本

为什么能提高收益率：

- 比赛最终比的是选出收益率更高的股票，而不是整体误差最小
- 如果目标函数更贴近 Top-K 选择，往往更容易提升组合收益

具体做法：

- 在 `train.py` 中新增：
  - `target_mode=topk_emphasis_rank`
  - 或 `target_mode=winsorized_rank`
- 与当前 `cross_section_rank` 做 walk-forward 对比
- 以 `cumulative_return_after_cost` 和 `sharpe_after_cost` 决定是否替换默认方案

### 2. 优先方向二：强化验证方式，避免“看起来不错，实际上不稳”

当前状态：

- 已有 walk-forward

下一步可尝试：

- 增加更多折数
- 改变验证窗长度
- 单独统计不同市场阶段下的收益表现

为什么能提高收益率：

- 真正能提高收益率的方案，不一定是某一折最好，而是跨时间窗更稳定
- 验证更稳，默认方案更不容易在线上翻车

具体做法：

- 把 `num_folds` 从 `3` 扩成 `4~5`
- 尝试 `valid_dates=10 / 15 / 20`
- 输出不同折的：
  - `Top5收益`
  - `Sharpe`
  - `回撤`

### 3. 优先方向三：继续优化推理阶段，而不是急着换模型

当前状态：

- 已经有风险过滤
- 已经有风险调整排序
- 已经有低换手执行

下一步最值得试的，是更细的推理控制：

- 候选池大小：`30 / 40 / 50`
- 风险过滤阈值更细粒度搜索
- 分层筛选：
  - 先按分数筛
  - 再按波动 / 换手 / 振幅二次过滤
- 对异常高波动、极端换手股票增加更强惩罚

为什么能提高收益率：

- 你目前最优收益率提升，很大一部分已经来自推理逻辑和换手控制，而不是换模型
- 这说明“推理层”仍然是收益率提升的重要杠杆

具体做法：

- 把当前 `compare_profiles` 扩成更细网格搜索
- 输出：
  - `top_k`
  - `candidate_size`
  - 风险阈值
  - 成本后收益
  - Sharpe
- 选出新的更优 profile 再冻结

### 4. 优先方向四：继续压换手，但不要盲目压到过低

当前状态：

- 当前平衡版默认配置为 `max_turnover=0.70`
- 从局部精修搜索结果看，`0.68 ~ 0.76` 区间都具有竞争力

下一步可尝试：

- `0.4`
- `0.45`
- `0.55`
- 增加“持仓保留阈值”逻辑

为什么能提高收益率：

- 你已经证明过高换手会明显吃掉成本后收益
- 但换手压得太低，也可能错过新的高收益机会
- 所以这一步本质是继续寻找“收益与成本”的平衡点

具体做法：

- 做一轮更细粒度 turnover 消融
- 尝试“若旧持仓仍在前 N 名，则优先保留”的机制
- 比较：
  - 收益率
  - 换手率
  - 夏普
  - 最大回撤

### 5. 优先方向五：补更多真正有区分度的特征

当前状态：

- 你已经有不错的基础、技术和风险特征

下一步更值得补的是：

- 行业内 / 横截面相对强弱特征
- 更长窗口趋势特征
- 收益率偏度 / 极值类特征
- 连续上涨 / 下跌天数特征
- 波动率状态切换特征

为什么能提高收益率：

- 当前消融说明“完整特征集”比精简特征集更好
- 这通常意味着特征空间还有继续做强的潜力

具体做法：

- 每次只新增一小组特征
- 保持同样回测与消融流程
- 不做大杂烩式加特征，避免难以解释

### 6. 暂时不建议作为第一优先级的方向

以下方向可以做，但不建议现在最先投入：

- 一上来就做复杂深度模型
- 大规模模型融合
- 过早引入太多外部框架

原因：

- 你当前最优结果主要是靠“标签、特征、推理、换手控制”抬起来的
- 说明当前收益瓶颈更可能在策略细节和验证稳健性，而不一定在模型容量不够

## 八、如果只允许做三件最有可能提高收益率的事

如果时间有限，我建议按这个顺序做：

### 第一件：做更细粒度的推理参数搜索

重点搜索：

- candidate size
- 风险阈值
- 排序惩罚权重
- turnover cap

重点尝试：

- `max_turnover=0.4 / 0.45 / 0.55`
- 旧持仓优先保留机制

因为你现在已经验证过低换手方向是有效的，继续精修这块很可能还有空间。

### 第三件：新增一小组更有区分度的风险/横截面特征

不要一下子加很多，而是小步快跑：

- 加一组
- 回测
- 消融
- 看是否超过当前默认方案
