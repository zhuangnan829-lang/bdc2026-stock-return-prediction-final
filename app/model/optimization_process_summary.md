# 收益率与稳定性优化总结文档

生成时间：2026-05-25  
项目：面向沪深 300 成分股的金融时间序列收益预测与组合推荐研究  
当前主线：LSTM sl20 + `base_alpha_v3_rs_crowding_mini4` + `cross_section_rank` + `risk_adjusted` 排序 + `pred` 权重 + Top5  

## 一、总览结论

本轮 Prompt 0-21 的核心成果不是简单“换一个模型”，而是把原来偏经验判断的模型选择流程，升级成了有统一配置、统一回测、统一排行榜、风险规则和完整验证流水线的实验体系。

最终判断如下：

- 不建议直接替换 LSTM sl20 主线。
- 原因是目前没有任何新方案能在 walk-forward、回测、单切片三方面同时、稳定地优于 sl20。
- 建议继续保留 LSTM sl20 作为主线。
- 同时保留 aggressive / robust 双配置：
  - aggressive 用于比赛冲分和收益优先场景。
  - robust 用于高波动震荡、低换手和风险展示场景。
- 当前 robust 观察候选为 `mt050_tc0010_blend_0.5_cap0.20`，它显著降低换手和回撤，但收益下降较多，因此适合“候选观察”，不适合直接替换主线。

完整验证流水线已执行，13 个关键步骤全部通过，失败步骤为 0。

## 二、主要提升

### 1. 工程可靠性提升

原先的问题是多个脚本、快照、配置文件之间可能存在口径不一致。现在通过统一配置读取、配置一致性检查、提交前检查和完整验证流水线，把“哪个配置才是当前正式默认”这件事明确下来了。

当前已具备：

- `default_submission_config.json` 作为主配置来源。
- `compare_config_consistency.py` 检查默认配置、best_config、model_meta、脚本默认值是否一致。
- `pre_submit_check.py` 做提交前完整校验。
- `run_full_optimization_validation.py` 串起 13 个关键验证步骤。

这意味着后续不会再只凭某一次输出文件判断结果，而是有一套可重复执行的验证流程。

### 2. 组合风险控制能力提升

组合层面已经明确验证了权重上限、pred/equal 混合、低换手约束的效果。

当前主线回测表现：

- 成本后累计收益：`1.171246`
- Sharpe：`4.019488`
- 最大回撤：`-0.090067`
- 平均换手：`0.956671`

权重上限实验显示：

- `cap=0.20` 单切片得分略高：`0.030099`
- 但累计收益略降至：`1.165010`
- `cap=0.18` 回撤改善到 `-0.087064`，换手降到 `0.931137`，但收益降到 `1.085936`

低换手压力测试显示：

- `mt050_tc0010_blend_0.5_cap0.20`
- 成本后收益：`0.653410`
- Sharpe：`4.053580`
- 最大回撤：`-0.048990`
- 平均换手：`0.500000`

结论：风险确实能降，尤其是换手和回撤；代价是累计收益明显下降。因此这类方案适合作为 robust 候选，而不是默认替换主线。

### 3. 模型选择从“凭感觉”升级为排行榜规则

已建立 `experiment_leaderboard.csv` 和 `experiment_leaderboard_report.md`，统一扫描实验目录，并使用 `stable_alpha_score` 综合评分：

`0.25 * top5_return_mean + 0.20 * cost_after_return + 0.20 * sharpe + 0.15 * rank_ic_mean + 0.10 * worst_fold_rank_ic - 0.05 * drawdown_penalty - 0.05 * turnover_penalty`

同时加入硬性淘汰规则：

- fold 内 Top5 最差收益小于等于 0，淘汰。
- worst fold RankIC 明显低于 sl20，谨慎或淘汰。
- 最大回撤明显恶化，淘汰。
- 换手升高且收益没有补偿，淘汰。
- 单切片高但 walk-forward 变差，标记疑似过拟合。
- result_validator 不通过，淘汰。

这个改动非常关键：后续不会因为某个模型单次收益高就贸然采用。

### 4. 明确识别了高风险候选

排行榜中部分 TopK 目标模型虽然回测收益很高，但被规则识别为高风险，例如：

- `topk20_gamma3_0`
- `topk30_gamma3_0`
- `topk20_gamma5_0`
- `topk30_gamma5_0`
- `v4_medium_lstm_sl40`
- `sl60`

主要原因包括：

- `top5_return_min_by_fold <= 0`
- worst fold RankIC 明显低于当前 sl20
- 最大回撤明显恶化

这说明新模型并非没有亮点，而是稳定性证据不足，不满足“全面替换主线”的条件。

### 5. 明确了市场阶段风险

市场阶段分析显示，高波动震荡阶段是当前主线的明显弱点：

- 低波动阶段 RankIC：`0.042727`
- 趋势阶段 Top5 收益：`0.014351`
- 高波动震荡阶段 RankIC：`-0.007992`
- 高波动震荡阶段 Top5 收益：`-0.006384`

这解释了为什么 robust 配置有存在价值：它不是为了追求最高累计收益，而是为了在不利阶段降低执行风险。

## 三、没有明显提升的地方

### 1. 新模型没有全面替代 sl20

根据最终验证规则，只有新模型同时在 walk-forward、回测、单切片三方面优于 sl20，才建议替换主线。当前没有满足这个条件的方案。

一些新模型或新目标在累计收益上有亮点，但 fold 稳定性、worst fold、Top5 最差收益或回撤不达标。

### 2. Rank blend 没有提升主线

Rank blend 结果显示，多个融合方案没有超过 baseline LSTM：

- baseline LSTM rank_ic_mean：`0.027982`
- `A_lstm70_lightgbm30` rank_ic_mean：`0.017071`
- `B_lstm60_lightgbm20_momentum20` rank_ic_mean：`0.016463`
- `D_lstm50_lightgbm25_xgboost25` rank_ic_mean：`0.001854`

这些融合没有改善 worst fold，反而降低了 RankIC 或 Top5 稳定性。因此目前不建议用 rank blend 替换主线。

### 3. 长序列和 Transformer-lite 暂不适合作为主线

sl40/sl60、Transformer-lite 等分支没有达到全面胜出标准。部分分支存在回撤恶化或 fold 内收益不稳定问题。

这符合原文档中的原则：Transformer-lite、sl60 或其他新模型只有全面超过 sl20，才允许考虑替换主线。

## 四、当前最终建议

### 默认主线

保留 LSTM sl20 主线。

理由：

- 当前主线收益、Sharpe、回测表现仍然强。
- 没有新模型在三方面同时胜出。
- 新模型中有不少疑似过拟合或 fold 不稳定信号。

### 比赛冲分配置

使用 aggressive 配置：

`aggressive_lstm_sl20_alpha_v3_rs_crowding_mini4_cs180_rp-30_pred_mt100_cap20`

特点：

- 宽候选池 `cs180`
- `pred` 权重
- `max_turnover=1.0`
- `max_single_weight=0.20`
- 追求单切片得分和累计收益

### 稳定策略配置

使用 robust 配置：

`robust_lstm_sl20_alpha_v3_rs_crowding_mini4_cs180_rp-30_blend05_mt050_cap20`

特点：

- `blend_0.5`
- `max_turnover=0.50`
- `max_single_weight=0.20`
- 更适合高波动震荡阶段和风险展示

### 是否同步默认配置

不建议自动同步。当前应继续保留默认配置不变，等最终提交目标明确后再手动确认：

- 如果目标是比赛冲分，考虑 aggressive。
- 如果目标是稳健展示，考虑 robust。

## 五、后续提升建议

### 1. 针对高波动震荡阶段单独优化

当前高波动震荡阶段表现最弱，应优先做阶段化处理：

- 在高波动震荡阶段启用低换手 robust 配置。
- 降低纯 pred 权重依赖。
- 引入反转保护、拥挤度惩罚和波动过滤。
- 单独评估高波动震荡阶段的 RankIC、Top5 收益和回撤。

### 2. 继续优化 Top-K 目标，但必须加稳定性约束

Top-K 加权目标能拉高收益上限，但部分候选 fold 内最差收益为负，说明目标可能过拟合。

建议下一步不是继续盲目提高 TopK gamma，而是加约束：

- 训练目标中加入 fold 稳定性惩罚。
- 对 `top5_return_min_by_fold <= 0` 的模型直接降权或淘汰。
- 用 worst fold RankIC 作为早停或模型选择指标之一。

### 3. 单票集中问题仍需继续处理

当前单票贡献集中仍是主要风险之一。建议继续做：

- 单票最大贡献占比监控。
- 单票权重上限动态化，而不是固定 0.18 或 0.20。
- 对历史贡献过高的股票加入拥挤度或回撤风险惩罚。

### 4. 换手控制可以作为模式开关，而不是默认常开

低换手配置能显著降低回撤和交易成本，但收益下降较多。

建议不要全局启用低换手，而是在以下场景启用：

- 高波动震荡阶段。
- 市场 RankIC 连续下降。
- 单票集中度异常升高。
- 当前预测分布置信度不足。

### 5. 融合模型要先解决“负迁移”

LightGBM、Momentum、XGBoost、Transformer-lite 融合暂未稳定超过 LSTM。后续可以考虑：

- 只在某些市场阶段启用融合。
- 用 stacking 选择器，而不是固定权重。
- 对每个模型先做阶段适配评估，再决定是否参与融合。

## 七、最终一句话

这轮优化最大的提升，是把项目从“追逐单次最高分”推进到了“可复现、可解释、可审计的模型选择体系”。收益上限确实看到了一些候选突破，但稳定性证据不足；因此当前最稳妥的策略是保留 LSTM sl20 主线，同时保留 aggressive/robust 双配置，后续重点攻克高波动震荡阶段、Top-K 目标稳定性和单票集中风险。
