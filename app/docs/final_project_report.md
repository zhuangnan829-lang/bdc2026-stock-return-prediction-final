# 沪深300股票收益预测与组合推荐结题报告

## 一、课题概述

本项目围绕沪深300成分股的短期收益预测与组合推荐展开，目标是在给定历史行情数据的基础上，构建可复现的数据处理、特征工程、模型训练、回测评估和提交生成流程，最终输出满足竞赛格式要求的五只股票等权组合 `result.csv`。

项目定位不是单一模型脚本，而是一条完整实验链路：从原始数据读取、行情特征构建、多模型对比、组合约束校验，到 Docker 彩排和自动化测试。当前最终提交方案为 `aggressive_score_submission`，对应输出文件已冻结在 `app/output/result.csv` 与 `test/output/result.csv`。

## 三、技术路线

项目采用“多模型候选 + 统一打分 + 组合约束”的实现思路。

数据处理阶段，系统读取训练集、测试集和股票列表，完成行情字段清洗、股票代码标准化、时间排序、缺失处理和滚动窗口特征构建。特征工程覆盖动量、波动率、成交量、趋势、反转、风险调整收益、市场状态等维度，并通过 preset 机制控制不同特征集合的规模与用途。

模型阶段，项目重点实现并记录了以下候选：

- Momentum baseline：作为可解释、稳定的传统基线。
- LightGBM：用于表格特征下的非线性排序建模。
- XGBoost：用于补充树模型对比和稳健性验证。
- LSTM：用于捕捉局部时间序列依赖。
- Transformer-lite：用于验证轻量注意力结构在有限样本下的表现。

组合生成阶段，系统根据候选股票得分筛选 Top 5，并按竞赛要求生成股票代码与权重字段。最终权重总和为 1，单只股票权重为约 0.2，满足提交格式要求。

## 五、模型选择说明

综合实验稳定性、实现复杂度、可解释性和提交结果表现，最终提交不采用单一深度模型直接输出，而是采用更稳健的候选打分与冻结提交流程。原因如下：

- 金融时间序列噪声高，深度模型在样本规模有限时容易出现局部过拟合。
- 树模型和动量类特征在本数据结构下具备更好的可解释性和调参效率。
- 竞赛提交只要求最终组合结果，稳定可复现比堆叠更多模型更重要。
- Docker 彩排和提交冻结要求产物一致，最终优先选择可稳定复现的方案。

因此，LSTM、LightGBM、XGBoost、Transformer-lite 和 Momentum baseline 主要承担实验比较、候选验证和方案论证角色，最终通过冻结提交脚本固化输出。

## 六、未纳入正式实验链路的方向

开题报告中提到的 ARIMA、Qlib、FinRL、TFT、N-HiTS、TSFM 等方向均已在材料中给出调研或适用性分析，但未全部纳入正式实验链路。主要原因包括：

- ARIMA 更适合单变量平稳序列，难以直接适配多股票横截面选股任务。
- Qlib 适合长期维护的大规模量化研究平台，本项目周期内接入成本高于收益。
- FinRL 偏强化学习交易环境，需要更完整的交易成本、持仓状态和交互式环境设定。
- TFT、N-HiTS、TSFM 对数据规模、训练资源和严格验证集设计要求更高，短周期内难以保证比当前方案更稳健。

相关说明见 `app/docs/unimplemented_model_applicability.md` 和 `app/docs/opening_report_alignment.md`。

## 八、结论与后续工作

本项目已经完成从股票行情数据到最终组合推荐结果的闭环实现，并补齐了评测反馈中指出的关键短板：测试失败、Docker 彩排不一致、交付口径不统一、模型承诺说明不足和 Git 证据偏弱等问题。当前版本具备代码可运行、结果可复现、材料可核验的交付条件。

后续若继续扩展，可重点开展三项工作：

- 引入更严格的滚动时间验证，减少单一切片偶然性。
- 将 Qlib 或 FinRL 作为独立研究分支接入，而不是混入当前提交主链路。
- 在更多市场阶段上比较深度模型与树模型的稳定性，形成更系统的模型适用边界分析。
