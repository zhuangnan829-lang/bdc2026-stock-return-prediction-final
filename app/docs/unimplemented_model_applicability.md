# 未纳入正式实验链路的模型与框架适用性说明

本文档用于补充开题报告中提到、但当前没有形成完整可复现实验链路的模型与框架说明。结论不是“没有考虑”，而是已经完成适用性判断，并基于数据规模、任务目标、复现约束和交付风险，决定不把这些方向纳入最终正式提交链路。

## 1. 总体结论

当前项目最终交付采用“统一特征工程 + walk-forward 验证 + Top-K 排序推荐 + 成本后组合回测 + Docker 复现”的闭环。已完整落地并进入同协议比较的方向包括：

- LSTM 主线
- LightGBM / XGBoost / Linear Regression / Momentum 基线
- Transformer-lite 候选分支
- rank blend、风险过滤、权重约束、市场状态分析等增强实验

开题报告中提到的 `ARIMA / TFT / N-HiTS / TSFM / Qlib / FinRL` 当前没有纳入正式实验链路，主要原因是它们与本项目最终评价目标存在不同程度的不匹配，或会显著增加依赖、复现和答辩解释成本。

## 2. 逐项说明

| 方向 | 开题中的定位 | 当前处理方式 | 未纳入正式实验链路的原因 | 后续可补充方式 |
|---|---|---|---|---|
| ARIMA | 传统统计模型 baseline | 调研与适用性分析 | ARIMA 更适合单变量时间序列点预测；本项目核心是沪深300横截面 Top-K 排序和组合推荐。若逐股票建模，需要为约300只股票分别拟合并维护阶数，且难以自然融合横截面相对强弱、拥挤度和风险过滤特征。 | 选取少量代表股票做单变量收益预测示例，并说明其与 Top-K 组合目标的差异。 |
| TFT | 深度时序模型 | 调研与后续方向 | TFT 需要较规范的多变量时序数据集、静态/动态协变量设计和较长训练调参周期。当前数据规模与交付周期下，强行接入会引入大量框架依赖，且不一定优于已经闭环的 LSTM sl20。 | 在统一特征表基础上构造小规模 TFT 数据集，先做离线对照，不直接替换主线。 |
| N-HiTS | 深度时序预测模型 | 调研与后续方向 | N-HiTS 强项是多步数值预测，而当前任务重点是股票横截面排序、Top-K 命中和组合回测。若直接预测收益点值，仍需额外设计排序和组合转化层，短期内不如 LSTM 主线闭环稳定。 | 做收益率多步预测对照，并额外评估 RankIC 和 Top-K，而不是只看 MAE/RMSE。 |
| TSFM / TimeGPT / TimesFM / Chronos | 时间序列基础模型方向 | 前沿方向调研 | TSFM 多依赖预训练模型、外部服务或较大运行环境；比赛/答辩要求离线可复现和代码归档。当前项目还需要对齐沪深300日频金融数据、横截面排序和组合回测，直接接入的工程风险高于收益。 | 作为专题页展示调用方式、适用边界、失败案例和未来迁移方案。 |
| Qlib | 量化研究参考框架 | 流程参考与适用性分析 | Qlib 自带数据层、日历、数据处理器、模型和回测协议。完整接入会与当前比赛数据格式和现有流水线重复，短期内反而降低可解释性；当前项目已自行实现特征、训练、验证、回测和结果校验闭环。 | 后续可把现有数据转换为 Qlib 格式，做一版独立框架复现实验。 |
| FinRL | 强化学习量化框架 | 流程参考与适用性分析 | FinRL 更适合连续交易环境、状态-动作-奖励建模和长期策略学习；本项目当前提交目标是固定测试切片的 Top-K 权重结果，强化学习环境设计成本较高，且很难在短周期内证明优于监督排序模型。 | 后续可构造组合调仓环境，比较 RL 策略与当前排序回测策略。 |

## 3. 为什么不强行补半成品实验

正式结题更看重“研究闭环是否可信”，而不是模型名称堆叠。若在最终阶段强行加入 ARIMA、TFT、N-HiTS、TSFM、Qlib 或 FinRL 的半成品实验，容易产生以下问题：

1. 实验口径无法与当前 LSTM 主线完全一致。
2. 缺少 Docker 复现或依赖安装证据。
3. 指标只停留在点预测误差，不能解释 Top-K 推荐和组合收益。
4. 答辩时难以说明为什么这些模型没有进入最终提交。
5. 可能干扰已经通过校验的正式提交链路。

因此，当前更稳妥的结题口径是：已完成广泛调研与适用性收敛，真正进入正式实验的是能够在统一协议下完成训练、验证、回测、提交和复现的模型。

## 4. 结题报告建议表述

可在结题报告“问题总结与后续改进方向”或“模型选择依据”中使用以下表述：

> 开题阶段我们提出了较宽的模型探索范围，包括 ARIMA、LightGBM、XGBoost、LSTM、Transformer、TFT、N-HiTS、TSFM 以及 Qlib / FinRL 等参考框架。实际推进中，我们将“能否形成统一可复现实验链路”作为筛选标准：只有能在相同数据切分、相同特征口径、相同排序推荐指标和相同组合回测协议下评估的方向，才纳入正式对比。最终，LSTM、LightGBM、XGBoost、Linear Regression、Momentum 和 Transformer-lite 形成了可复现实验结果；ARIMA、TFT、N-HiTS、TSFM、Qlib 和 FinRL 当前保留为调研与后续扩展方向，未强行纳入正式提交，以避免引入不可复现或不可比较的半成品实验。

## 5. 答辩问答口径

如果被问到“为什么开题写了这些模型但没有全部做完”，可以回答：

> 我们没有把模型清单当成必须堆满的列表，而是按统一实验协议筛选。金融收益预测最终看的是横截面排序、Top-K 推荐和成本后组合回测，不只是单变量点预测误差。ARIMA、TFT、N-HiTS、TSFM、Qlib 和 FinRL 都有价值，但在当前数据、时间和复现要求下，强行接入会降低实验可信度。因此我们把它们作为适用性分析和后续方向，把正式结论建立在已经完成闭环的 LSTM、树模型、线性/动量基线和 Transformer-lite 对照上。
