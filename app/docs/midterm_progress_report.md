## 五、数据处理与特征工程进展

### 1. 数据来源与任务定义

项目围绕竞赛提供的沪深300相关历史行情数据展开，核心输入包括：

- `app/data/stock_data.csv`
- `app/data/hs300_stock_list.csv`
- `app/data/train.csv`
- `app/data/test.csv`

当前标签构造口径为：

- `future_open_1`
- `future_open_5`
- `target_return = (future_open_5 - future_open_1) / future_open_1`

这意味着任务不是预测任意时点价格，而是在赛题口径下，围绕 `T+1` 开盘买入、`T+5` 开盘卖出的未来收益进行建模。

### 2. 防止时间泄漏的处理原则

项目在设计上严格遵循时间序列任务要求：

- 训练集、验证集与测试集按交易日顺序划分
- 标签仅使用未来窗口收益，特征仅使用调仓日及以前可获得信息
- walk-forward验证采用滚动时间切分，而非随机切分
- 序列模型仅利用历史窗口构造样本，避免未来信息泄漏

当前正式默认配置采用：

- `valid_dates = 20`
- `num_folds = 3`
- `sequence_length = 10`

### 3. 当前正式特征体系

当前冻结的正式特征集为 `base_alpha_v3_rs_crowding_mini4`，共20个特征，包含收益、动量、振幅、量价关系与相对强弱等信息。当前模型元数据中记录的特征如下：

### 3. walk-forward验证结果

根据当前 `model_meta.json` 记录，walk-forward三折验证结果汇总如下：

| 指标 | 当前结果 |
|---|---:|
| RMSE 均值 | 0.498469 |
| MAE 均值 | 0.495315 |
| RankIC 均值 | 0.011680 |
| Top5平均收益均值 | 0.010119 |

各折结果显示：

- Fold 1：`rank_ic = -0.060918`
- Fold 2：`rank_ic = 0.036048`
- Fold 3：`rank_ic = 0.059911`

这说明当前模型在不同时段上的稳定性仍有波动，但整体已经形成正向的排序能力，并具备继续通过特征与执行层优化提升稳健性的空间。

进一步诊断后，当前正式默认方案已切换为 `sequence_length = 20` 的 `sl20` 版本。在相同协议下，`sl20` 的 `rank_ic_mean` 提升至 `0.027982`，但第一折仍为负，具体为 `fold_1 = -0.033492`、`fold_2 = 0.118946`、`fold_3 = -0.001508`。这说明将序列窗口从 10 扩展到 20 可以明显缓解“历史视野过短”问题，但并未完全消除阶段性不稳定。

针对短期噪声特征的进一步实验表明，问题并不是某一个单独特征失效，而更像是多列短期信号的联动扰动。单独去掉 `ret_1d`、`ret_3d`、`intraday_return`，以及去掉 `ret_1d + ret_3d` 或 `ret_3d + intraday_return`，都没有让第一折优于当前正式 `sl20`。只有同时去掉 `ret_1d`、`ret_3d`、`intraday_return` 三列时，第一折 `rank_ic` 才从 `-0.033492` 改善到 `-0.011686`，但整体 `rank_ic_mean` 会回落到 `0.007495`，说明稳定性改善是以牺牲整体排序能力为代价换来的。

基于第 1 折逐票诊断结果，可以进一步看到问题主要集中在一批短线活跃股票上。例如 `300502`、`000807`、`601689` 等股票，在多日样本中都表现为：真实收益较强，同时 `ret_1d`、`ret_3d`、`intraday_return` 三列特征值偏强，但当前正式 `sl20` 往往会将其排得过后；而删除三列短期特征后，这些股票的排名明显前移。与此同时，也存在 `601377`、`603019` 等股票在删除三列后被错误压低排名的情况。这说明短期特征并非完全无效，而是在特定阶段对部分活跃股票形成了不稳定联动，导致模型对“短线强势票”和“短线噪声票”的区分能力下降。

因此，当前更合理的结论是：模型稳定性问题并不单纯来自 LSTM 架构本身，也不只是单一短期特征失效，而更可能与市场阶段变化下短期收益、三日收益和日内收益三类信号的组合使用方式有关。后续优化方向应继续围绕阶段划分、短期特征组合约束以及排序层稳健性增强展开。
