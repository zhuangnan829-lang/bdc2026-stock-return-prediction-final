# 数据清单说明

## 1. 数据来源

- 数据平台：`baostock`
- 使用目的：本地训练、本地验证、特征工程、标签构造，以及后续比赛提交链路开发
- 当前状态：原始行情数据和沪深300成分股文件均已准备完成，并存放在 `app/data` 目录下

## 2. `app/data` 目录文件说明

- `stock_data.csv`
  - 角色：完整原始行情主数据
  - 说明：保留当前沪深300股票池的历史日频行情
- `hs300_stock_list.csv`
  - 角色：成分股股票池文件
  - 说明：保存沪深300成分股代码及名称
- `train.csv`
  - 角色：从 `stock_data.csv` 切分得到的本地训练集
  - 时间范围：`2024-01-02` 到 `2026-03-06`
- `test.csv`
  - 角色：从 `stock_data.csv` 切分得到的本地验证 / 预测集
  - 时间范围：`2026-03-09` 到 `2026-03-13`

## 3. 股票池与时间范围

- 股票池：沪深300成分股
- `stock_data.csv` 股票数量：`300`
- `hs300_stock_list.csv` 股票数量：`300`
- 原始数据总时间范围：`2024-01-02` 到 `2026-03-13`
- 训练集时间范围：`2024-01-02` 到 `2026-03-06`
- 本地验证集时间范围：`2026-03-09` 到 `2026-03-13`
- 成分股清单更新日期：`2026-03-16`

## 4. 字段说明

`stock_data.csv`、`train.csv`、`test.csv` 使用如下字段：

| 字段名 | 含义 |
|---|---|
| `股票代码` | 6位股票代码 |
| `日期` | 交易日期 |
| `开盘` | 开盘价 |
| `收盘` | 收盘价 |
| `最高` | 最高价 |
| `最低` | 最低价 |
| `成交量` | 成交量 |
| `成交额` | 成交额 |
| `振幅` | 振幅 |
| `涨跌额` | 涨跌额 |
| `换手率` | 换手率 |
| `涨跌幅` | 涨跌幅 |

`hs300_stock_list.csv` 使用如下字段：

| 字段名 | 含义 |
|---|---|
| `updateDate` | 成分股清单更新日期 |
| `code` | baostock 风格股票代码，例如 `sh.600000` |
| `code_name` | 股票名称 |

## 5. 股票代码和日期约定

- 原始行情中的股票代码必须按字符串读取，并在程序中统一标准化为 6 位字符串
- `hs300_stock_list.csv` 中的成分股代码为 `sh.600000` 这类 baostock 风格，匹配行情时需截取最后 6 位
- `stock_data.csv` 中的原始日期格式类似 `2024-01-02 00:00:00`
- `train.csv` 和 `test.csv` 在本地切分后使用 `YYYY-MM-DD` 格式

## 6. 复权口径

- 复权方式：后复权
- 来源依据：根据 `get_stock_data_illustration.txt`，抓取工具使用 `adjustflag="1"`，会自动处理除权除息相关调整

## 8. 开发说明

- `stock_data.csv` 是保留的原始主数据，不应手工修改
- `train.csv` 和 `test.csv` 仅用于本地开发、训练和验证
- 后续特征工程必须避免未来信息泄漏，只能使用当日及以前数据
- 标签构造必须严格遵循比赛口径：`T+1` 开盘买入，`T+5` 开盘卖出
